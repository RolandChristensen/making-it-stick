Question

In PHP, what do you use for a single-line comment and what do you use for a multi-line comment.

Answer

There are two single-line comments:

  1. # comment
  2. // comment

Multi-line comments use /* comments */

Question

In JavaScript, what do you use for a single-line comment and what do you use for a multi-line comment.

Answer

For a single-line comment: // comment

Multi-line comments use /* comments */

Question

Do Java and JavaScript share the same commenting tokens?

Answer

Yes, both use the same tokens to indicate a comment.

For a single-line comment: // comment

Multi-line comments use /* comments */

Question

In HTML, what do you use for a single-line comment and what do you use for a multi-line comment?

Answer

Trick question! HTML only has one way to comment code: <!-- comments -->

Trying to comment out a large section of HTML that contains other comments can be tricky, due to this limitation.

Question

In order to debug HTML or hide templated code, how can you simulate commenting out large sections of code containing other comments?

Answer

Create a CSS class that will hide a section of HTML and surround the HTML you want hidden with a section that has the class attribute used to hide it.

  <style>
    .hide-section {
      display: none;
    }
  </style>
  <section class="hide-section">
    <!-- Hide both this comment and paragraph. -->
    <p>Hide this paragraph.</p>
  </section>

Notes On Flashcard Topics

The "Single Line Comment"

Not all languages have a short hand for a single line comment. HTML and CSS do not have a shorthand for a single line comment, but many high level languages, such as C# and Python do.

  • The C's (C#, C++, and C): // Single Line Comment.
    • Slightly interesting: MS has coding conventions about multi-line comments. They use double forward slashes on each line to create multi-line comments. The Visual Studio IDEs have a multi-line comment feature, where you select the code you want to comment and click a button to comment all selected text. It does this by using a single-line comment on each line instead of a multi-line comment. /* imagine multiple lines */
  • CSS: /* Comment Here */ There is no single line comment. Use the same method for multi-line comments.
  • HTML: <!-- Single Line Comment -->
    • Bonus: To display the comment above on a web page, use the HTML entities for the less than and greater than symbols.
    • &lt;!-- Single Line Comment --&gt;
    • Guess how I got the less than and greater than entities to display on the page. :)
  • Java: // Single line comment.
  • JavaScript: // Single line comment.
  • PHP: There are two different string literals you can use for a PHP single line comment.
    • // Single Line Comment.
    • # Single Line Comment.
  • Python: # Single Line Comment.
  • SQL: -- Single Line Comment.

Multi-line Comments

In general multi-line comments start with some token such as (/*) and end with a token that closes the comment, such as (*/).

  • Some prefer to use a single line comment to comment multiple lines. For example:
    • // Line One.
    • // Line Two.
  • The C's (C#, C++, and C): /* Imagine multiple lines */
  • CSS: /* Imagine multiple lines */
    • /*
    • Line one.
    • Line two.
    • */
  • HTML: <!-- Imagine multiple lines -->
    • <!--
    • Line one.
    • Line two.
    • -->
  • Java: /* Imagine multiple lines */
  • JavaScript: /* Imagine multiple lines */
  • PHP: /* Imagine multiple lines */
  • Python: """ Imagine multiple lines """
  • SQL: /* Imagine multiple lines */
