Enumerators Flashcards

Question

What is the default value of an enumerator?

Answer

The default value is the first value in the list.

Question

How would you use an enumerator to check if a variable has been set or not?

Answer

Use the default value and set it to "Undefined" to indicate it has not been set. Use switch statements in code and have a "case" statement for "Undefined" for error checking.

Question

What exception should you use in a switch "default" section to indicate the value passed was not handled in the "case".

Answer

Use a ArgumentOutOfRangeException to indicate the value is out of the range of values in the "case."

Question

How would you create an enumerator to use in place of a boolean value.

Why would you do this?

Answer

public enum IsGoodIdea
{
    Undefined,
    False,
    True
}
Use this so you know if the variable has been set or not.

Question

Write a switch to handle the enumerator from the previous flashcard. Use a switch statement.

Answer

switch (isGoodIdea)
{
    case IsGoodIdeaEnum.False:
        return DoThis();
    case IsGoodIdeaEnum.True:
        return DoThat();
    case IsGoodIdeaEnum.Undefined:
        throw new ArgumentException("Parameter not set.");
    default:
        throw new ArgumentOutOfRangeException(
          "The switch did not have a case for the value.");
}

Question

Write a switch to handle the enumerator from the previous flashcard. Use a switch expression.

Answer

return isGoodIdea switch
{
    IsGoodIdeaEnum.False => DoThis(),
    IsGoodIdeaEnum.True => DoThat(),
    IsGoodIdeaEnum.Undefined => throw new ArgumentException(
      "Parameter not set."),
    _ => throw new ArgumentOutOfRangeException(
      "The switch did not have a case for the value.")
};

Notes On Flashcard Topics

Enumerators

Use enumerators to outline a list of known values for a variable. For example a list of chairs: recliner, office, folding, etc.. Rather than using a string, which could introduce errors use an enumerator to enforce valid values. A good example I use professionally, is an enumerator for the environments code goes through before making it into production. 

public enum EnvironmentEnum
{
  Undefined,
  Dev,
  Test,
  Prod
}
Start each enumerator with the default value. If the variable is not set, it will default to the first in the list. In this case it will be "Undefined" and we can throw errors if it should be set but was not. This is a good argument for using an enumerator instead of the default bool. Boolean in lots of languages only has two values true or false and most of them default to false. You cannot tell if the variable has never been set if you get a false, you have to assume the value is an actual false. If you use an enumerator, however, you will know if it has not been set or if the value is meant to be false. 
public enum IsChairEnum
{
  Undefined,
  True,
  False
}
It is usually a good idea to use enumerators in conjunction with switch statements that hold all of the possible values of the switch. The switch helps to identify any errors in the future if you add or subtract values from the enumerator. The compiler will throw errors if you remove a value from an enumerator and you will flag errors at runtime if you add a value to the enumerator and forget to check all the places it is used. The "default" of the "case" will throw an exception if none of the "case" statements match the value passed. 
switch (environment)
{
    case EnvironmentEnum.Dev:
        return new Uri("https://authservice.dev.com/");
    case EnvironmentEnum.Test:
        return new Uri("https://authservice.test.com/");
    case EnvironmentEnum.Prod:
        return new Uri("https://authservice.Prod.com/");
    case EnvironmentEnum.Undefined:
        throw new ArgumentException(
            "The parameter was not set before the switch.");
    default:
        throw new ArgumentOutOfRangeException(
            "The switch did not have a case for the value.");
}
C# Switch Expression 

return environment switch
{
    EnvironmentEnum.Beta => new Uri("https://service.beta.com/"),
    EnvironmentEnum.Test => new Uri("https://service.test.com/"),
    EnvironmentEnum.Prod => new Uri("https://service.Prod.com/"),
    EnvironmentEnum.Undefined => throw new ArgumentException(
        "The parameter was not set before the switch."),
    _ => throw new ArgumentOutOfRangeException(
        "The switch did not have a case for the value.")
};
