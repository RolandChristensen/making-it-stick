Notes On Input for Apps

Input for small mobile apps should take the size and input of the device into account. The primary input of a mobile device will the the touch screen. Avoid using the keyboard if possible. Instead of a text box for numeric input use a slider, for instance.

App Design

  • Ask: Who is the end user?
  • Ask: What are the main objectives of the app?
  • Instead of adding features, ask about the behaviours the user will want to do with the app?
  • How will you organize the pages for easy navigation?
  • The core functionality should be on the first page user navigates to.
  • Other functions should be in a menu for easy access.
  • Consider supported devices. Test all devices supported.
  • Easy to read colors and font theme.
  • Provide feedback for each user action. Long running tasks need an animation. Non-reversible actions, like credit card transactions should be clearly outlined. Errors should have clear instructions to resolve the issue.
  • Use common design choices, like a hamburger menu at the top of the page, so people understand how to use your app quickly and easily.

Do you remember your Celsius to Fahrenheit?
celsius = (fahrenheit - 32) * 5/9
fahrenheit = celsius * 9/5 + 32
In the normal range of temperatures we usually experience, fahrenheit is higher than celsius. At extreme lows fahrenheit starts to be a larger negative number. They are equal at -40.

Speed of light: 299,792,458 m/s
velocity: m/s
acceleration (The rate of change of velocity): m/s^2
Initial Velocity: u
final velocity: v
acceleration: a
time: t
distance: s
a = (v-u)/t
t=(v-u)/a
v=u+a*t
s=u*t+1/2*a*t^2
gravitational acceleration: g=9.81m/s^2 (going down) g=-9.81m/s^2 (going up)

Algorithms

  1. A sequence of unambiguous steps.
  2. When these steps are executed, the algorithm should produce correct results for all possible cases/inputs.

Example:
Input: Velocity u, Angle θ
Known constant: Gravitational acceleration g=-9.81 m/s^2
Output: height and distance
Algoritm:
  1. Calculate the horizontal velocity ux=u*cos(θ)
  2. Calculate the vertical velocity uy=u*sin(θ)
  3. Calculate the time until the projectile reaches its maximum height. t=(v-uy)/a=(0-uy)/g=(0-uy)/-9.81=-uy/-9.81=uy/9.81
    tmaxHeight=uy/9.81
  4. Calculate the time until the projectile lands (twice tmaxHeight) - tlanding=2*t
  5. Height is calculated using s=u*t+1/2*a*t^2
    height=uy*tmaxHeight-1/2*9.81*tmaxHeight^2
  6. Distance is calculated using s=u*t+1/2*a*t^2, where a = 0, t=tlanding, u = ux
    distance=ux*tlanding

JavaScript to calculate height and distance of projectile

 function calculate(angle, velocity) {
  var horizontalVelocity = velocity*Math.cos((angle*Math.PI)/180);
  var verticalVelocity = velocity*Math.sin((angle*Math.PI)/180);
  var timeToMaxHeight = verticalVelocity/9.81;
  var timeToLanding = 2*timeToMaxHeight;

  var distance = calculateDistance(horizontalVelocity, timeToLanding);
  var height = calculateHeight(verticalVelocity, timeToMaxHeight);
}

function calculateDistance(horizontalVelocity, time) {
  var distance=horizontalVelocity*time;
  return distance;
}

function calculateHeight(verticalVelocity, time) {
  var height = (verticalVelocity*time) - (0.5*9.81*time*time);
  return height;
}

Local Storage

Local Storage is part of the Window Object
It stores data with no expiration date.
The data is not deleted when the browser is closed, and is available for future sessions unlike session storage which is deleted on window close.
Storage Limit: 5MB (much larger than cookies)
Consider using local storage to avoid latency when calling an api.
Security: data is not sent, so is not exposed to data sniffers.
Each site has it's own storage area, which guarantees that a website can access only data stored by itself.
The user can change browser settings to never store anything locally, in which case a SecurityException will be thrown.

function drawCircle(context, centerX, centerY, radius) {
  // Book says always start with .beginPath(), but w3schools showed it was not neccessary for the line above.
  // I will need to read more documentation to figure out when it is needed or not.
  context.beginPath();

  // .arc() takes five parameters. The first three are self documented.
  // The last two are the beginning arc in radians and the final arc in radians.
  // For a circle 0 is the beginning arc and 2π (360°) the final.
  context.arc(centerX, centerY, radius, 0, 2 * Math.PI);
  context.stroke();
}
