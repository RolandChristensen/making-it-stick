Testing

After uploading your page to the internet, check the functionality in various browsers to ensure functionality across all supported browsers.

On failure: determine if it is your problem or a browsers problem. If it is the browser, consider removing the functionality or the browser from the list of supported browsers.

Notes On Input for Apps

Input for small mobile apps should take the size and input of the device into account. The primary input of a mobile device will the the touch screen. Avoid using the keyboard if possible. Instead of a text box for numeric input use a slider, for instance.

App Design

  • Ask: Who is the end user?
  • Ask: What are the main objectives of the app?
  • Instead of adding features, ask about the behaviours the user will want to do with the app?
  • How will you organize the pages for easy navigation?
  • The core functionality should be on the first page user navigates to.
  • Other functions should be in a menu for easy access.
  • Consider supported devices. Test all devices supported.
  • Easy to read colors and font theme.
  • Provide feedback for each user action. Long running tasks need an animation. Non-reversible actions, like credit card transactions should be clearly outlined. Errors should have clear instructions to resolve the issue.
  • Use common design choices, like a hamburger menu at the top of the page, so people understand how to use your app quickly and easily.

Slider Input Widget

Use slider to avoid using the keyboard on a mobile. Note that it is best to use this when exact values are not important. It may be hard to get the slider to respond in even units of measurement, depending on the size of the range.

  • Range Slider: a slider with the min and max range set.
  • Input element: set the type="range" attribute. Displays a slider.

HTML

Input element: creates input element such as button, text box, checkbox, etc... Each has a lable tag associated to it via the "for" attribute of the label and the "name" attribute of the input element.

  • type attribute: there are many types of input (button, checkbox, color, date, etc..)
  • name attribute: used to find the input element in the DOM.
    Only form elements with name attributes will be sent when "submitting" a form.
  • id attribute: used to find the attribute in the DOM, by the label element for instance via the "for" attribute. Ideal for ensuring unique id's for finding an element. An HTML checker will flag any duplicate id's on a single page.
  • min attribute: for number and date inputs to indicate the lowest value accepted.
  • max attribute: for number and date inputs to indicate the highest value accepted.
  • placeholder attribute: the text in the field to prompt the user what is accepted by the input field. Works for text, search, url, tel, email, and password inputs.

Do you remember your Celsius to Fahrenheit?
celsius = (fahrenheit - 32) * 5/9
fahrenheit = celsius * 9/5 + 32
In the normal range of temperatures we usually experience, fahrenheit is higher than celsius. At extreme lows fahrenheit starts to be a larger negative number. They are equal at -40.

Validating input with event handlers.
The 'blur' event fires when an element has lost focus.
The opposite of 'blur' is 'focus'.
There is an optional parameter 'useCapture' that defaults to false.
False means the handler is executed in the bubbling phase.
True means the handler is executed in the capturing phase.
Search for 'Event dispatch and DOM even flow'
Bubbling goes up through the DOM tree from the element with the listener. Capture phase goes down to the children of the element with the listener.

function initialize() {
  var someInput = document.getElementById("someId");
  someInput.addEventListener('blur', validateInput);

UX

It is good to keep in mind that the most frequent tasks should have the fewest clicks to perform. In the Fahrenheit to Celsius app the most common task for people in the US would be to convert from Fahrenheit to Celsius, so that is the default. An extra click is required to do the inverse.
Everything is clearly labeled with °C or °F and it is easily grasped what each control does.
It is also on a single page, so you can take it in all at once without any navigation.
Always provide feedback as soon as possible after any interaction by the client.
Mobile device buttons should be large enough to be easily tapped.
