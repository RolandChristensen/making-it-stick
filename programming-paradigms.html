Question

Is SQL an "Imperative Programming Language" or a "Declarative Programming Language"?

Answer

SQL is a declarative programming language.

Question

Briefly describe "Declarative Programming"

Answer

Declarative programming: declaration statements are used to describe the logic needed to get results. A programmer will declare the logic expected and is not concerned with how the calculations are done to get the results. Example: SELECT * FROM TableName WHERE value-x = 0. This declares that we expect all records in the table where value-x equals 0. There are no loops, lists, or return statements, just a declarative statement.

Notes On Flashcard Topics

Declarative Programming Paradigm

The opposite of imperative programming, declarative programming focuses on the outcome rather than the steps required to achieve the outcome. In other words, declarative programs expresses the logic without describing the control flow. The programmer declares an object with specific properties not specific directions about how this will be accomplished. Example: I want a pizza with thin crust and olive oil base and mozzarella, but I do not specify how the dough is made or what order the items should be added.

Structured Query Language (SQL), used in relational databases, is an example of a declarative language. Most of the time you will declare what you want from the database: SELECT * FROM PizzaTable WHERE Crust = 'thin'. The computation is conducted and all pizzas are checked if they are thin crust, but we do not describe the computation in code. We don't tell it to use a foreach, but it most certainly checks all pizzas in the table.

Declarative programming is an umbrella term that applies to a group of programming paradigms including "Functional Programming" and "Logic Programming".
